title: "Push notification API reference for CleverTap"
description: "Reference for CleverTapAPI push methods: FCM token management, notification creation, channel setup, permission requests, click tracking, and bitmap utilities."
---

The `CleverTapAPI` class exposes a set of static and instance methods for every aspect of push notification handling — registering device tokens, building and rendering notifications, managing Android notification channels, requesting runtime permission on Android 13+, and tracking delivery and click events. Most notification-creation and channel methods are static and do not require a live instance.

---

## Token methods

### `fcmTokenRefresh`

<Warning>Deprecated since core v5.1.0. Use `tokenRefresh` from your `onNewToken` implementation instead.</Warning>

```java
public static void fcmTokenRefresh(Context context, String token)
```

Iterates over all available `CleverTapAPI` instances and refreshes the FCM token for each non-analytics-only instance.

<ParamField path="context" type="Context" required>
  Android application context.
</ParamField>
<ParamField path="token" type="String" required>
  The FCM registration token to refresh.
</ParamField>

### `getDevicePushToken`

```java
public String getDevicePushToken(PushType type)
```

Returns the cached device push token for the given `PushType`, or `null` if not yet available. On initial install the token may not be available immediately — implement `DevicePushTokenRefreshListener` to receive a callback when the token arrives.

<ParamField path="type" type="PushType" required>
  The push provider type whose token to retrieve.
</ParamField>

<ParamField path="pushType" type="PushType" required>

</ParamField>

## Notification creation

### `createNotification(Context, Bundle)`

```java
public static void createNotification(Context context, Bundle extras)
```

Launches an asynchronous task that downloads the notification icon from CleverTap and renders the Android notification. Use this when implementing your own FCM handling.

<Note>Starting from core v5.1.0 this method runs on the caller's thread. Call it inside `onMessageReceived()` of your messaging service.</Note>

<ParamField path="context" type="Context" required>
  Android application context.
</ParamField>
<ParamField path="extras" type="Bundle" required>
  The `Bundle` received by the FCM broadcast receiver.
</ParamField>

<CodeGroup>

```java Java
@Override
public void onMessageReceived(RemoteMessage message) {
    Bundle extras = mParser.toBundle(message);
    CleverTapAPI.createNotification(getApplicationContext(), extras);
}
```

```kotlin Kotlin
override fun onMessageReceived(message: RemoteMessage) {
    val extras = mParser.toBundle(message)
    CleverTapAPI.createNotification(applicationContext, extras)
}
```

</CodeGroup>

## Permission methods

### `isPushPermissionGranted`

```java
public boolean isPushPermissionGranted()
```

Checks whether the POST_NOTIFICATIONS runtime permission is granted. Returns `true` if granted, `false` if denied. Relevant for Android 13 (API 33) and above.

<CodeGroup>

```java Java
boolean granted = cleverTapAPI.isPushPermissionGranted();
```

```kotlin Kotlin
val granted = cleverTapAPI.isPushPermissionGranted()
```

</CodeGroup>

---

### `promptForPushPermission`

```java
public void promptForPushPermission(boolean showFallbackSettings)
```

Directly invokes the OS runtime notification permission dialog, bypassing the push primer in-app. For Android 13+ only.

<ParamField path="showFallbackSettings" type="boolean" required>
  If `true` and the permission has already been denied, shows an alert dialog directing the user to the app's notification settings page. If `false`, logs a verbose message instead.
</ParamField>

<CodeGroup>

```java Java
cleverTapAPI.promptForPushPermission(true);
```

```kotlin Kotlin
cleverTapAPI.promptForPushPermission(true)
```

</CodeGroup>

---

### `promptPushPrimer`

```java
public void promptPushPrimer(JSONObject jsonObject)
```

Displays a local in-app push primer before requesting the OS notification permission. Build the `JSONObject` configuration using `CTLocalInApp.builder()`. See the [in-app notification API](/api/in-app-notification-api) for the full `CTLocalInApp.builder()` reference.

<ParamField path="jsonObject" type="JSONObject" required>
  Configuration object produced by `CTLocalInApp.builder()`.
</ParamField>

<CodeGroup>

```java Java
JSONObject jsonObject = CTLocalInApp.builder()
    .setInAppType(CTLocalInApp.InAppType.HALF_INTERSTITIAL)
    .setTitleText("Get Notified")
    .setMessageText("Please enable notifications to stay up to date.")
    .followDeviceOrientation(true)
    .setPositiveBtnText("Allow")
    .setNegativeBtnText("Cancel")
    .setBackgroundColor("#FFFFFF")
    .setBtnBorderColor("#1C84FE")
    .setTitleTextColor("#1C84FE")
    .setMessageTextColor("#000000")
    .setBtnTextColor("#FFFFFF")
    .setBtnBackgroundColor("#1C84FE")
    .build();

cleverTapAPI.promptPushPrimer(jsonObject);
```

## Bitmap utilities

Both methods must be called on a **background thread**. They return `null` if the download fails, times out, or the network is unavailable.

### `getNotificationBitmapWithTimeout`

```java
public static @Nullable Bitmap getNotificationBitmapWithTimeout(
    Context context,
    Bundle bundle,
    String bitmapSrcUrl,
    boolean fallbackToAppIcon,
    long timeoutInMillis
)
```

Downloads a notification bitmap with a configurable timeout. Use this for custom notification rendering.

<ParamField path="context" type="Context" required>
  Android application context. Must not be `null`.
</ParamField>
<ParamField path="bundle" type="Bundle" required>
  The push payload `Bundle` received from FCM. Must not be `null`.
</ParamField>
<ParamField path="bitmapSrcUrl" type="String" required>
  URL of the image to download.
</ParamField>
<ParamField path="fallbackToAppIcon" type="boolean" required>
  If `true`, returns the app icon when the bitmap cannot be downloaded.
</ParamField>
<ParamField path="timeoutInMillis" type="long" required>
  Download timeout in milliseconds. Must be between 1 and 20000 (inclusive).
</ParamField>

<CodeGroup>

```java Java
@Override
public void onMessageReceived(RemoteMessage message) {
    Bundle bundle = mParser.toBundle(message);
    // Must be on a background thread
    Bitmap bitmap = CleverTapAPI.getNotificationBitmapWithTimeout(
        context, bundle, "https://example.com/icon.png", true, 5000);
}
```

```kotlin Kotlin
override fun onMessageReceived(message: RemoteMessage) {
    val bundle = mParser.toBundle(message)
    // Must be on a background thread
    val bitmap = CleverTapAPI.getNotificationBitmapWithTimeout(
        context, bundle, "https://example.com/icon.png", true, 5000)
}
```

</CodeGroup>

Downloads a notification bitmap with a configurable timeout and maximum file size. Returns `null` if the bitmap exceeds `sizeInBytes`.

<ParamField path="context" type="Context" required>

</ParamField>
<ParamField path="bitmapSrcUrl" type="String" required>
  URL of the image to download.
</ParamField>
<ParamField path="fallbackToAppIcon" type="boolean" required>
  If `true`, returns the app icon when the bitmap cannot be downloaded.
</ParamField>
<ParamField path="timeoutInMillis" type="long" required>
  Download timeout in milliseconds. Must be between 1 and 20000 (inclusive).
</ParamField>
<ParamField path="sizeInBytes" type="int" required>
  Maximum size of the bitmap in bytes. Must be greater than 0.
</ParamField>

<CodeGroup>

```java Java
@Override
public void onMessageReceived(RemoteMessage message) {
    Bundle bundle = mParser.toBundle(message);

```

```kotlin Kotlin

    val bundle = mParser.toBundle(message)
```

</CodeGroup>
