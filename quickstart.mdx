---
title: "Get started with CleverTap Android SDK"
description: "Install and integrate the CleverTap Android SDK step by step — from adding the Gradle dependency and configuring credentials to sending your first event."
---

This guide walks you through installing the CleverTap Android SDK, configuring your credentials, setting up the lifecycle callback, and sending your first event. By the end you will have a working integration ready for analytics and user engagement.

<Steps>
  <Step title="Add the SDK dependency">
    Declare the SDK in your module-level `build.gradle` file:

    ```groovy build.gradle
    dependencies {
        implementation "com.clevertap.android:clevertap-android-sdk:8.1.0"
    }
    ```

    The SDK is published to `mavenCentral` as an AAR. Make sure `mavenCentral()` and `google()` are listed under `repositories` in your project-level `build.gradle`.
  </Step>

  <Step title="Add required dependencies">
    Add the Firebase Messaging library and AndroidX Core to your module-level `build.gradle`:

    ```groovy build.gradle
    dependencies {
        implementation "com.clevertap.android:clevertap-android-sdk:8.1.0"
        implementation "androidx.core:core:1.13.0"
        implementation "com.google.firebase:firebase-messaging:24.0.0"

        // Required only if you enable Google Ad ID collection (off by default)
        implementation "com.google.android.gms:play-services-ads:23.6.0"
    }
    ```

    Also include the `google-services` classpath in your project-level `build.gradle` and apply the plugin at the bottom of your module-level `build.gradle`:

    ```groovy project build.gradle
    buildscript {
        repositories {
            google()
            mavenCentral()
        }
        dependencies {
            classpath "com.android.tools.build:gradle:8.9.3"
            classpath "com.google.gms:google-services:4.4.2"
        }
    }
    ```

    ```groovy module build.gradle
    apply plugin: 'com.google.gms.google-services'
    ```

    Add your FCM-generated `google-services.json` to your app module directory.
  </Step>

  <Step title="Add CleverTap credentials to AndroidManifest.xml">
    Add your CleverTap account credentials inside the `<application>` tag in `AndroidManifest.xml`. You can find these values in your CleverTap dashboard under **Settings → Integration → Account ID, SDKs**.

    ```xml AndroidManifest.xml
    <meta-data
        android:name="CLEVERTAP_ACCOUNT_ID"
        android:value="Your CleverTap Account ID" />
    <meta-data
        android:name="CLEVERTAP_TOKEN"
        android:value="Your CleverTap Account Token" />
    ```
  </Step>

  <Step title="Set up the lifecycle callback">
    CleverTap requires an `Application` lifecycle callback to track notification opens, display in-app notifications, handle deep links, and record important user behavior.

    **Option A — Use the CleverTap Application class** (simplest):

    Set `android:name` on the `<application>` tag in `AndroidManifest.xml`:

    ```xml AndroidManifest.xml
    <application
        android:label="@string/app_name"
        android:icon="@drawable/ic_launcher"
        android:name="com.clevertap.android.sdk.Application">
    ```

    **Option B — Use a custom Application class**:

    If you already have a custom `Application` class, call `ActivityLifecycleCallback.register(this)` before `super.onCreate()`:

    <CodeGroup>

    ```java MyApplication.java
    public class MyApplication extends Application {
        @Override
        public void onCreate() {
            ActivityLifecycleCallback.register(this);
            super.onCreate();
        }
    }
    ```

    ```kotlin MyApplication.kt
    class MyApplication : Application() {
        override fun onCreate() {
            ActivityLifecycleCallback.register(this)
            super.onCreate()
        }
    }
    ```

    </CodeGroup>

    <Note>
      This step is required. Without it, CleverTap cannot track notification opens, display in-app notifications, or handle deep links correctly.
    </Note>
  </Step>

  <Step title="Get the default instance">
    After the lifecycle callback is registered, retrieve the default CleverTap instance anywhere in your app:

    <CodeGroup>

    ```java MainActivity.java
    CleverTapAPI clevertap = CleverTapAPI.getDefaultInstance(getApplicationContext());
    ```

    ```kotlin MainActivity.kt
    val clevertap = CleverTapAPI.getDefaultInstance(applicationContext)
    ```

    </CodeGroup>

    The SDK creates a shared singleton based on the account credentials in `AndroidManifest.xml`. The method returns `null` if credentials are missing, so guard against that in production code.
  </Step>

  <Step title="Send your first event">
    Push a simple event to verify the integration is working:

    <CodeGroup>

    ```java MainActivity.java
    clevertap.pushEvent("App Launched");
    ```

    ```kotlin MainActivity.kt
    clevertap.pushEvent("App Launched")
    ```

    </CodeGroup>

    After running the app, check your CleverTap dashboard. The event should appear within a few seconds under **Events**.
  </Step>
</Steps>

## Next steps
